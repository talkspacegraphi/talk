# Приоритетные улучшения Vortex Messenger
## Для использования ~10 человек

**Дата:** 2026-04-26

---

## 🎯 Критичные улучшения (сделать в первую очередь)

### 1. Производительность сообщений
**Проблема:** Все сообщения загружаются в DOM сразу, при большой истории будет тормозить.

**Решение:**
- Пагинация: загружать по 50 последних сообщений
- Подгружать старые при скролле вверх
- Виртуализация списка (react-window или react-virtuoso)

**Файлы:**
- `apps/web/src/components/ChatView.tsx`
- `apps/server/src/routes/messages.ts`

---

### 2. Индексы базы данных
**Проблема:** Запросы к сообщениям будут медленными при росте истории.

**Решение:** Добавить индексы в Prisma schema:
```prisma
model Message {
  // ...
  @@index([chatId, createdAt])
  @@index([senderId])
}

model ChatMember {
  // ...
  @@index([userId])
}
```

**Файл:** `apps/server/prisma/schema.prisma`

---

### 3. Сжатие изображений
**Проблема:** Большие фото занимают место и долго загружаются.

**Решение:**
- Сжимать на клиенте перед отправкой (browser-image-compression)
- Генерировать thumbnails на сервере (sharp)
- Ограничить максимальный размер (2MB)

**Файлы:**
- `apps/web/src/components/MessageInput.tsx`
- `apps/server/src/routes/messages.ts`

---

### 4. Резервное копирование БД
**Проблема:** SQLite файл может повредиться, потеряются все данные.

**Решение:**
- Автоматический бэкап раз в день
- Хранить последние 7 копий
- Скрипт восстановления

**Создать:** `apps/server/scripts/backup.ts`

---

### 10. Валидация размера файлов
**Проблема:** Нет проверки размера загружаемых файлов на сервере.

**Решение:**
- Ограничить размер файла (10MB для фото, 50MB для видео)
- Проверка на сервере через multer limits
- Понятное сообщение об ошибке

**Файл:** `apps/server/src/routes/messages.ts`
